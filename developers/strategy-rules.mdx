Use this page when building `POST /strategies` or `PATCH /strategies/:id` payloads programmatically.

This reference is aligned with:

- backend external API DTO validation
- frontend rule catalog used by the Strategy Wizard

## Rule object schema

A rule in `dealbreakers` and `ruleGroups[].rules` uses the same shape:

```json
{
  "id": "rule-1",
  "ruleType": "liquidity_usd",
  "operator": "greater_than",
  "value": 10000,
  "enabled": true
}
```

Field requirements:

- `id`: required string
- `ruleType`: required string
- `operator`: required string, one of:
  - `greater_than`
  - `less_than`
  - `equals`
  - `not_equals`
  - `is`
  - `is_not`
- `value`: required (`string | number | boolean`)
- `enabled`: required boolean

## Group schema

Rule groups are required on create.

For UI compatibility, use these four standard groups from the Strategy Wizard tabs:

- `Security`
- `Market`
- `Socials`
- `AI`

Use these canonical group IDs:

- `security`
- `market`
- `socials`
- `ai`

Canonical shape:

```json
{
  "id": "market",
  "name": "Market",
  "logic": "AND",
  "rules": []
}
```

Group fields:

- `id`: required string
- `name`: required string
- `logic`: required (`AND` or `OR`)
- `rules`: required array of rule objects

<Warning>
Do not use arbitrary group names like `Rules` if you want payloads to round-trip cleanly through the current UI tabs. Stick to `Security`, `Market`, `Socials`, and `AI`.
</Warning>

## Strategy-level fields often missed

When creating or updating a Strategy, include these strategy-level fields as needed:

- `strict` (boolean): data strictness mode
- `alertCooldownMins` (number, optional): cooldown per token in minutes, valid range `0` to `1440`

Cooldown behavior:

- After an alert is sent for token `X`, this Strategy suppresses new alerts for token `X` until cooldown expires.
- Other tokens are unaffected.
- In the UI this is shown as **Alert Cooldown (mins)**.

## Dealbreakers vs grouped rules

- `dealbreakers`: hard requirements checked first
- `ruleGroups[].rules`: grouped rules evaluated after dealbreakers

Both use the exact same rule object shape.

## Operator and value compatibility

Use the operator that matches the `valueType` of the `ruleType`.

| `valueType` | Allowed operators | `value` type |
| --- | --- | --- |
| `boolean` | `is`, `is_not` (only where specified in catalog) | `true` or `false` |
| `number` | `greater_than`, `less_than` | number |
| `percentage` | `greater_than`, `less_than` | number |
| `string` | `equals`, `not_equals` | string |

<Warning>
The API validates `operator` globally, but correct evaluation depends on pairing each `ruleType` with the right operator and value type from the catalog below.
</Warning>

## AI preset mapping

In the frontend UI, validity presets map to numeric ranges. If you are calling the API directly, send range rules explicitly:

- `very_relevant` -> `greater_than: 80` and `less_than: 100`
- `somewhat_relevant` -> `greater_than: 50` and `less_than: 100`
- `any` -> omit both rules

Example:

```json
{
  "ruleType": "website_content_validation_score",
  "operator": "greater_than",
  "value": 80,
  "enabled": true,
  "id": "rule-min"
}
```

```json
{
  "ruleType": "website_content_validation_score",
  "operator": "less_than",
  "value": 100,
  "enabled": true,
  "id": "rule-max"
}
```

## End-to-end payload example

Example create payload with mixed boolean, range, enum, and AI rules:

```json
{
  "name": "API Strategy Example",
  "description": "Built from integration service",
  "strict": true,
  "alertCooldownMins": 30,
  "retryPolicy": {
    "type": "QUICK_RETRY",
    "maxAttempts": 3,
    "delayMinutes": 15
  },
  "matching": {
    "enabled": true,
    "minPercent": 80
  },
  "triggers": [
    {
      "id": "trigger-1",
      "eventType": "new_token_created",
      "enabled": true
    }
  ],
  "dealbreakers": [
    {
      "id": "db-1",
      "ruleType": "is_honeypot",
      "operator": "is",
      "value": false,
      "enabled": true
    },
    {
      "id": "db-2",
      "ruleType": "risk_level",
      "operator": "not_equals",
      "value": "CRITICAL",
      "enabled": true
    }
  ],
  "ruleGroups": [
    {
      "id": "market",
      "name": "Market",
      "logic": "AND",
      "rules": [
        {
          "id": "r1",
          "ruleType": "liquidity_usd",
          "operator": "greater_than",
          "value": 10000,
          "enabled": true
        },
        {
          "id": "r2",
          "ruleType": "market_cap",
          "operator": "less_than",
          "value": 1000000,
          "enabled": true
        }
      ]
    },
    {
      "id": "ai",
      "name": "AI",
      "logic": "AND",
      "rules": [
        {
          "id": "r3",
          "ruleType": "website_content_validation_score",
          "operator": "greater_than",
          "value": 80,
          "enabled": true
        },
        {
          "id": "r4",
          "ruleType": "website_content_validation_score",
          "operator": "less_than",
          "value": 100,
          "enabled": true
        }
      ]
    }
  ],
  "isActive": false
}
```
